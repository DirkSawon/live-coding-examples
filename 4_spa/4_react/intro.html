Was ist eine Single Page Application?

Single Page Applications sind einezelne seiten, die viele informationen liefern können, und spezifische inhalbe aktualisieren ohne die seite wechseln oder neu laden zu müssen.

Wenn wir beispielsweise auf GMAIL sind, und uns durch die emails und ordner klicken, sehen wir das sich ausser des hauptinhaltes nichts ändern. Also die seite läd nicht neu, nur spezifische inhalte werden aktualisiert.

SPA Architektur

Die SPA ist eine webanwendung oder "Webseite", die mit dem benutzer interagiert, indem sie aktuelle seiten dynamisch umschreibt, anstatt neue seiten vom server zu laden.

Dieser ansatz minimiert die unterbrechung der benutzererfahrung zwischen den aufeinanderfolgenden seiten, so das die anwendung sich mehr wie eine desktop anwendung, also ein "computerprogramm" verhält, weswegen es logisch ist uns web-entwickler und nicht web-designer zu nennen.

Auf den meisten webseiten gibt es viele sich wiederholende inhalte, einige davon bleiben immer gleich, egal wo der benutzer sich hinbewegt, zum beispiel der header, der footer, die navigationsleiste, uns so weiter - Allgemein bleiben viele elemente des layouts gleich.

Single Page Applications machen sich die wiederholungen von elementen, sogenannten komponenten zu nutze, indem man komponenten anlegt, und wiederverwenden kann.

Bei traditionellen Multi Page Applications, würden wir bei jeder seite die wir anfragen, immer die GANZE seite, mit allen inhalten geliefert kriegen.

Bei einer SPA hingegen, bekommt unser browser eine bauanleitung, wie er die seite zusammensetzen soll, und wo sich was wiederholt, oder benutzt wird, um das alles zusammenzufügen.

Die geschwindigkeit der SPA kommt dadurch zu stande, das ab diesem zeitpunkt, nur noch einzelene informationen ausgetauscht werden müssen, aber keine webseite mit all ihren daten mehr komplett heruntergeladen werden muss, während bei der klassischen seite jetzt immernoch bei jeder anfrage, die seite mit allen inhalten neu geladen werden muss.

Da bei SPA's die aufgabe des zusammensetzens der webseite auf seiten des browsers liegt kann der inhalt dynamisch neu geschrieben werden, anstelle komplet neu geladen werden zu müssen.

Anstelle das wir die seite immer neu laden, sehen wir also nur die neuen ersetzten inhalte.

Gerade im JavaScript bereich sind SPA's beliebt weil man schnell und einfach komplette webseiten, oer web-applikationen erschaffen, und diese dan wie eine desktop applikation nutzen kann.

Das konzept MVC

MVC steht für Model-View-Controller, eine speziele strukutur für web applikationen, in der die applikation in 3 komponenten eingeteilt wird, die voneinander getrennt entwickelt werden können.

  • Controller: Die steuerung der applikation, hier sitzt die programmlogik, die die daten und die sichtbare webseite verbindet
  • Model: Die datenbankanbindung, die mit dem controller verbunden ist, um daten auszutauschen
  • View: Die darstellung der daten auf der HTML seite, die über den controller zum benutzer kommen

Der vorteil dieser flexiblen art der programmierung liegt in der einfachen wiederholbarkeit, und wartung. Im team ist es auch einfacher an MVC's zu arbeiten, wenn jeder seiten bestimmten aufgabenbereich hat. Allerdings ist das setup um eine MVC aufzubauen komplizierter und man ist möglicherweise abhängig von modulen anderer entwickler.

Was ist React?

React ist eine JavaScript framework bibliothek von FaceBook (Meta), genau wie bootstrap von Zurb Twitter kommt, hat Facebook also mit React einen großen einfluss auf das gehabt, was wir als modernes web kennen.

React wurde entwickelt um interaktive benutzeroberflächen zu erstellen. Der hauptvorteil von React ist, das man sehr einfach verschiedene views oder einzelen kleine teile, genannt Komponenten, erstellen kann, die auf anfrage einfach geupdated werden können. Jeder komponent ist in der lage, seine eigenen daten und strukturen zu managen, weswegen man sie fantastisch zu komplexen benutzeroberflächen zusammenfügen kann.

Da die applikation aus einzelnen komponenten besteht, kann man sie voneinander getrennt updaten oder warten, ohne die gesuchte stelle auf tausenden zeilen html oder javascript suchen zu müssen.

React hat eine sogenannte state und prop verwaltung, das sind 2 wichtige konzepte von React.

  • Props sind eigenschaften die von aussen an komponenten übergeben werden um die funktionalität zu erweitern, diese properties können innerhalb des komponenten verwenden werden. Das ist vergleichbar mit parametern die wir an eine funktion übergeben.
  • States sind eigenschaften die innerhalb einer komponente sind, um einfluss auf den zustand des views haben zu können. Diese states müssen auf spezielle art und weise gesetzt, genutzt und verändert werden, damit sie in echtzeit dynamisch verwendet werden können. Dies ist auch der grund, wieso react, react heisst, denn wenn sich ein staste innerhalb einer komponente verändert, wird automatisch die renderfunktion aufgerufen, was dazu führt das die neuen daten direkt angezeigt werden, es wird also auf einen state reagiert... reacted.

React arbeitet ausserdem mit sogenannten hooks, dies sind die states der verschiedenen lebensphasen eines komponenten, wie zum beispiel den zeitpunkt des ladens des komponenten, den wir damit direkt managen können, ausserdem gibt es hooks, die wir direkt einspeisen können um dynamisch den view zu verändern.

Fangen wir an mit React

Die einfachste möglichkeit am anfang mit react zu arbeiten, ist das terminal programm "create-react-app", das uns automatisch ein rudimentäres react projekt erstellt, mit dem wir arbeiten können. Create-react-app können wir entweder mit NPM installieren, oder die live-version nutzen, in dem wir npx, also den live tracker von npm benutzen.

Wir nutzen npx, damit wir npx auch mal im einsatz sehen.

Wir erstellen einen ordner, gehen mit dem terminal hinein. und schreiben "npx create-react-app" und fügen am ende unseren projektnamen ein. (npx create-react-app first-app)

Wie wir sehen installiert es react und einige andere module, also warten wir mal ab.

In der zwischenzeit können wir mal in den browser, und uns ein chrome plugin namens "React developer tools" installieren, das wir in zukunft öfter mal benutzen wrden, um uns unsere komponenten im detail anzuschauen.

Wir geben bei google "React-developer tools" ein, klicken das gesuchte ergebnis auf der chrome web store an und installieren es im browser.

create-react-app hat uns jetzt eine komplette applikation erstellt, die wir theoretisch einfach ausführen können, mit den befehlen die uns aus der package.json im terminal bereit gestellt werden, aber schauen wir uns erst einmal an, was wir jetzt haben.

Einige dateien und ordner wurden erstellt.

Der node-modules ordner ist riesig wie erwartet, hier ist alles drin, was wir zum starten con react brauchen, wen wir unsere applikation erweitern, brauchen wir gegebenenefalles noch andere module, aber für den anfang reicht das, was wir haben.

Außerdem haben wir jetzt die ordner public und src.

in public finden wir die haupt html datei, die die app ausführen wird, und alles, was sonst noch zu der html datei gehört, wie zum beispiel das favicon.

Im ordner src finden wir unser eigentliches react projekt

  • App.css beinhaltet unser CSS, das wir für die applikation grundsätzlich anlegen
  • App.js ist unsere eigentlich applikation, dies ist ein beispiel komponent
  • App.test.js beinhaltet tests für die erfolgreiche ausführung unserer applikation
  • index.css beinhaltet css für den body, und vordefinierte elemente
  • index.js ist die datei, die unsere app in der html aufruft und beides miteinander verbindet.

Den rest können wir ignorieren, und uns mal die applikation anschauen.

Wir schreiben jetzt "npm run start" und sehen den live server von create-react-app laufen.

Wenn wir jetzt in der App.js änderungen machen, sehen wir, durch den live server diese direkt.
